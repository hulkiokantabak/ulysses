The Design Record

Base document 06 · written 18 July 2026, the day the author commissioned design. The sequence held to the end: documentation (00–04), architecture (05), the unstyled site under contract — and only then the look. This document records how the look was decided, what it is, and what it refuses, so that it can be re-opened deliberately rather than drifted from.

How it was decided

  1. Panel session 02 (panel-session-02-design.md): seven advisors, three rounds, binding vetoes. Direction D, Addresses Before Paint carried at 12, grafting A, The Sealed Quotation (9). Two directions died under argued vetoes — the cinematic threshold, and the empty matrix as front door.
  2. A design swarm produced three complete stylesheet candidates under the ratified brief — Folio (the book page), Ledger (Ithaca as design logic), Restraint (near-defaults minimalism) — each with a computed contrast manifest.
  3. Three judges — the Shneiderman floor (who recomputed every contrast claim and verified rendered behaviour in a browser), the Kiberd first-reader, and a joint Braudel/Warburg furniture-and-plate lens — scored them Folio 25.5 · Ledger 20.5 · Restraint 17, with itemized defects.
  4. A synthesis pass repaired every judged defect in Folio and grafted the losers' named strengths. The result is design/site.css, deployed at /assets/site/site.css.

What it is

  • The reading room. A 65ch serif column (Iowan Old Style / Palatino / Georgia stack), 17px at reading width, 1.65 leading, on paper #f4efe2 with deep #081a35 ink. The site's core surfaces are twenty-minute documents; the page is built for them. Minor-third scale; no webfonts — a system stack is both faster and free of fabricated provenance.
  • The mono factual layer (ui-monospace stack, 14px floor): status lines, the visible canonical address on every page, hours, captions, matrix labels, empty-state labels, and tables — whole tables, because "table numerics in mono" cannot be guaranteed cell-wise for unknown future markdown.
  • One new deep plate: the 16×4 dimension/register matrix on /dimensions/ — deep ground, hard edges, visible whole on load, keyboard-walkable (focus ring drawn inside the scroll box at outline-offset:-2px, Ledger's graft, so edge cells can never clip it). Cell state rides in semantic elements plus the literal word, never colour alone (Restraint's graft): <strong>built</strong> on the site's only gold, <em>building</em>, plain planned, <del>dropped</del>, <s>skip</s>, —.
  • Gold discipline: #dfa93f exists only on deep ground (8.17:1). On paper, gold's form is #A0701C and it never carries a letterform — it is the focus ring, the link underline, the aria-current bar (≥3:1 as a UI indicator; 3.79:1 measured).
  • The quotation grammar, installed before any dimension needs it: blockquote + cite + mono .q-address; quoted matter is never justified, hyphenated, clamped, ellipsized or case-transformed. Senn's red line is now a stylesheet fact.
  • The two artifacts are byte-verbatim assets (/assets/characters/…, /assets/day/…), embedded in orientation frames — the earlier head-injection of a canonical is gone; the frame pages carry the canonical and the way back.
  • Print keeps the footer, drops only nav and iframes, restores wide tables to full display, and renders the built state as a #A0701C border (gold's print form per 00-method.md).

What it refuses (as binding as what it has)

Photography and imagery · network webfonts · icons in the chrome · animation and transitions · hover-only reveals · ambient ash/rose washes (those colours belong to Stephen and Molly, who are not on these pages yet) · gold letterforms on paper · serif display type for schema titles · uppercase performance · empty-state theatrics · any rule keyed to today's emptiness.

Two author-directed exceptions, recorded 18 July 2026: the practice's shared GoatCounter page-view counting (cookieless, no personal data — the one external request, allow-listed exactly in tools/check.mjs), and the home-screen icons and web manifest (the 1922 object as an app icon: paper U on deep ground, one gold rule — design/icons/), both as on the practice's other sites.

The deck form (presentation register, added 19 July 2026)

The first audience deck (Four Odysseys) establishes the deck form, applying the ratified 1922 system rather than opening a new design decision — the panel's system extends to it. An audience deck is a self-contained HTML card-viewer on the deep plate (the register's slides are projected surfaces, so deep ground, like the two interactive artifacts): image cards only, no explanatory prose, keyboard-navigable (arrows, Home/End, dot tablist), an aria-live caption, and no animation — so reduced-motion is honoured by construction. Cards carry identity labels (a work's name, year, and its reading on the chart), never the argument; the argument is the presenter deck, kept in presenter-decks/ in the private repository and never passed through to the site (author's ruling, C-19). A deck is an artifact with register presentation, embed: false (a slideshow wants full width, so the dimension page links it rather than embedding it), and it surfaces automatically on /presentation/ and /registers/presentations/.

Enforcement

tools/check.mjs inverted at this commit — from forbidding style to protecting words: exactly one stylesheet per page and no stray styling vectors; rendered document text sampled and verified against source on all twelve document pages; artifacts and PDFs byte-compared to their sources; every declared colour pairing in data/design.json recomputed against its WCAG floor at every build (with a standing sanity assertion that gold-on-paper fails, so the forbidden pairing can never be declared conforming); fragment links resolve against real heading ids.

Implementer's additions beyond the synthesis (recorded per the handback discipline)

  • The matrix cell semantics in site/dimensions/index.njk map grid states to the semantic elements the synthesis specified; a dropped dimension's cells read dropped regardless of budget.
  • The 404 page links the stylesheet directly (it does not use the base layout).
  • The synthesis's .artifact-frame + iframe rule set width but no height, leaving the embedded artifacts at the 150px iframe default; a height: max(78vh, 34rem) was added so the plate is a room, not a slot.
  • The .artifact-embed class on the iframes is inert (the stylesheet targets .artifact-frame + iframe); it is left in place as a future hook.

The user-cohort tests

Three pre-registered personas — a first-time reader, an experienced Joycean, and an accessibility professional — tested the designed site in isolated cycles; an evaluator per cohort triaged findings across the content/presentation boundary (bounded presentation fixes applied; content findings queued for the author; anything that would erode an honesty marker refused). Results and the applied fixes are appended below.

Appendix — the cohort verdicts, the applied fixes, the author queue and the refused list are recorded in docs/playtest-01.md. All three cohorts would return; the two failures every cohort named — the artifact letterbox and the stale "unstyled" claim — were fixed the same day.
