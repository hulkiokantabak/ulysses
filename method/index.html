How this is being made

The project's working documents are published, including its record of its own errors. Documentation first, design second: the base documents were built while the book was fresh; the site ran deliberately unstyled until the documentation and architecture were complete (anchors A1, A18), and the paper-and-ink look now on the page was then commissioned by the author, decided in panel session 02, and applied on 18 July 2026. The shape of the presentation is still to come.

  • The four output registers — HTML, hyperlinked PDF, the Madrid-logic presentation, and diagrams; with the shared visual system and the standing rules. Base document 00.
  • The dimensions working document — the seven types of operation and the sixteen dimensions (titled "The Fifteen" before dimension 16 was recorded), with the anchors A1–A18. Base document 04.
  • The advisors and their sessions — the nine-advisor build panel, the masters session that stress-tested the dimensions, and the panel session that decided the build order.
  • The design record — base document 06: how the look was decided, what it is, and what it refuses.
  • The log — catches and learnings: every error found, with severity and what changed because of it.
  • The coverage research — what already exists on every angle this project considered, checked before any novelty claim.
